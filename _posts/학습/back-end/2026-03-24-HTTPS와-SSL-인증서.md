## 모바일 앱에서 HTTP는 안 된다

[TasteNote](/posts/로컬-개발환경-분리와-도메인/)가 토스 미니앱이다. 모바일 앱에서 API를 호출하려면 HTTPS가 필수. iOS의 ATS(App Transport Security), Android의 Network Security Config 모두 HTTP 요청을 기본 차단한다. HTTPS가 없으면 API가 먹통.

## HTTP vs HTTPS

HTTP는 평문 전송이다. 비밀번호를 적은 엽서를 우체부한테 맡기는 거다. 누구든 배달 중간에 들여다볼 수 있다. HTTPS는 봉투에 넣어서 봉인. TLS로 암호화한다.

| | HTTP | HTTPS |
|---|---|---|
| 포트 | 80 | 443 |
| 암호화 | 없음 | TLS |
| 주소창 | "주의 요함" | 자물쇠 |
| 모바일 앱 | 기본 차단 | 정상 |

## SSL 인증서

HTTPS를 쓰려면 SSL 인증서가 필요하다. 두 가지 역할.

1. **신원 증명**: "이 서버가 진짜 이 도메인이 맞다"는 보증
2. **암호화 키 교환**: 브라우저와 서버가 안전한 통신 채널을 만드는 데 사용

## 인증서를 받는 법

### Let's Encrypt (직접 설치)

무료. `certbot`으로 발급. 90일마다 갱신 필요. 서버에서 80포트로 도메인 인증을 받아야 한다. 직접 관리해야 할 게 있다.

### Cloudflare (자동)

Cloudflare 프록시를 사용하면 인증서가 자동 발급·갱신된다. 서버에 certbot이나 인증서 설치가 필요 없다. [Cloudflare Tunnel](/posts/홈서버-배포-인프라-Docker-보안-Cloudflare-Tunnel/)을 쓰고 있으니까 이 방식을 선택.

## 구간별 프로토콜

```
브라우저 ←HTTPS→ Cloudflare ←Tunnel→ cloudflared ←HTTP→ Nginx ←HTTP→ Spring Boot
```

| 구간 | 프로토콜 | 안전? |
|------|---------|------|
| 브라우저 ↔ Cloudflare | HTTPS | O (TLS) |
| Cloudflare ↔ cloudflared | Tunnel | O (터널 자체가 암호화) |
| cloudflared ↔ Nginx | HTTP (localhost) | O (같은 서버 내부) |
| Nginx ↔ Spring Boot | HTTP (localhost) | O (같은 서버 내부) |

서버 내부는 localhost 통신이라 HTTP여도 안전하다. 외부 구간만 암호화되면 된다. 집 안에서 가족끼리 하는 대화를 암호화할 필요 없는 거다.

## SSL 모드

Cloudflare Dashboard에서 SSL/TLS 모드를 설정한다.

| 모드 | 브라우저↔CF | CF↔서버 | 설명 |
|------|-----------|---------|------|
| Off | HTTP | HTTP | 암호화 없음 |
| Flexible | HTTPS | HTTP | CF가 HTTPS 처리. 서버는 HTTP |
| Full | HTTPS | HTTPS | 서버에도 인증서 필요 |
| Full (Strict) | HTTPS | HTTPS | 유효한 인증서 필요 |

Cloudflare Tunnel을 쓰면 터널 자체가 암호화돼 있으니까 Flexible로도 충분하다. 서버에 인증서를 설치할 필요 없다.

## 결론

Cloudflare Tunnel을 쓰면 HTTPS 관련해서 직접 할 일이 없다. 인증서 발급, 갱신, 설치 전부 자동. [DNS](/posts/DNS-레코드-타입과-동작-원리/)와 [Nginx](/posts/도메인-DNS-Nginx-리버스-프록시/) 설정만 하면 끝.
