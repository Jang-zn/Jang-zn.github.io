[앱인토스-통합 Api] 홈서버 배포와 CI/CD 구축

By 코드대장간
Posted Mar 20 Updated Mar 20 7 min read

배포 인프라 하루 만에

어제 API 포맷을 통일했고, 오늘은 배포 인프라를 통째로 구축했다. Docker 환경, CI/CD 파이프라인, Discord Webhook 연동까지. PR 2개 머지.

Docker 환경 전면 구축 — PR #13

멀티스테이지 Dockerfile

JDK 21에서 빌드하고 JRE 21에서 실행하는 2단계 빌드. ARG MODULE로 tastenote-api, admin-api를 같은 Dockerfile에서 빌드한다. 멀티모듈 프로젝트라 이게 필요했다.

docker-compose — 10개 서비스

한 서버에 prod/dev를 동시에 돌린다. 같은 건물에 1층은 실제 영업하는 매장, 2층은 테스트용 매장을 운영하는 셈이다.

환경서비스포트
Prodtastenote-api, admin-api8080, 8081
Devdev-tastenote-api, dev-admin-api8180, 8181
SharedMySQL, Nginx, Prometheus, Grafana-
캐시redis-prod, redis-dev6379, 6380

MySQL init 스크립트로 prod/dev 데이터베이스 자동 생성. Spring 프로필도 application-prod.yml, application-dev.yml로 분리해서 환경별 DB, Redis, 로그 레벨을 다르게 설정.

Nginx 리버스 프록시

서브도메인별 라우팅. nginx.conf.template + envsubst로 도메인을 환경변수로 주입. api.{DOMAIN} → prod, dev-api.{DOMAIN} → dev.

CI/CD — GitHub Actions + GHCR

master push → latest 태그로 GHCR push → SSH 접속 → prod만 restart. dev push → dev-latest 태그 → dev만 restart.

빌드는 matrix로 두 모듈을 병렬 처리. 배포 후 docker image prune으로 미사용 이미지 정리.

인프라 코드리뷰에서 잡은 것 3개

Claude가 만든 인프라 설정을 리뷰하다가 3개를 잡았다.

  1. MySQL healthcheck 인증 누락 — 인증 없이 mysqladmin ping만 하면 실제 준비 안 됐어도 healthy 뜬다. -u root -p$$MYSQL_ROOT_PASSWORD 추가. CMD를 CMD-SHELL로 바꿔야 환경변수 치환이 된다
  2. Nginx server_name api.* — 유효하지 않은 와일드카드. api.${DOMAIN}으로 템플릿화
  3. GHCR github.actor — fork PR 시 다른 사용자로 로그인 시도. secrets.GHCR_USERNAME으로 교체

자세한 건 CI/CD 학습 포스트에 정리.

Discord Webhook 연동 — PR #14

infra-discord 모듈

Clean Architecture 포트 패턴으로 구현. core-domain에 DiscordNotificationPort 인터페이스, infra-discord에 어댑터 구현. 포트 패턴 학습 포스트에 설계 배경 정리.

3종류 알림: 사용자 피드백, 서버 에러, 콘텐츠 신고. 각각 VO로 분리.

서버 에러 알림

GlobalExceptionHandler에서 500 에러 발생 시 ServerErrorEvent를 발행한다. ServerErrorEventListener가 비동기로 받아서 Discord로 전송. ApplicationEvent 패턴으로 예외 처리기와 알림 로직을 분리했다. 알림 실패가 에러 응답에 영향을 주지 않는다.

안정성 보강

첫 구현 후 두 가지를 추가 수정했다.

  • Webhook URL 로그 마스킹 — URL에 토큰이 포함되어 있어서 로그에 그대로 찍으면 안 된다. 앞 4자만 남기고 마스킹
  • 입력 검증 — 피드백 API 레벨(category 50자, title 200자, description 2000자)과 Discord Embed 레벨(SHORT 256자, LONG 1024자) 2단계 검증. Rate limiting도 분당 30건

피드백 엔드포인트

POST /api/v1/feedback — DB에 저장하지 않고 Discord로 바로 전송. 초기 단계에서 CS 시스템을 만드는 건 과하다. Discord 채널이 알림, 검색, 스레드를 다 해준다.

Claude Code 협업

Docker 환경 구축은 Claude한테 “홈서버에 prod/dev 동시 배포할 수 있게 Docker 환경 만들어줘”라고 지시. Dockerfile, docker-compose, nginx, CI/CD까지 한번에 나왔다. 양은 많았지만 구조가 깔끔했다.

다만 인프라 설정에서 실수 3개를 잡은 건 내 리뷰다. MySQL healthcheck 인증, Nginx 와일드카드, GHCR 로그인 — 전부 “돌아가는 것처럼 보이지만 특정 조건에서 터지는” 류의 버그다. Claude가 Docker/CI 설정을 빠르게 만들어주는 건 좋은데, 인프라 설정은 한 글자 실수로 전체가 안 돌아가니까 반드시 사람이 리뷰해야 한다.

Discord 모듈은 “Clean Architecture 포트 패턴으로 만들어줘”라고 했더니 포트/어댑터 분리, @Async, ApplicationEvent까지 한번에 구현. webhook URL 마스킹은 내가 “URL에 토큰 있는데 로그에 그대로 찍히면 안 돼”라고 짚어서 추가한 거다.

오늘의 결과

항목상태
Docker 멀티스테이지 Dockerfile완료
docker-compose 10서비스 (prod/dev/shared)완료
Nginx 리버스 프록시 (서브도메인 라우팅)완료
Spring 프로필 분리 (prod/dev)완료
GitHub Actions CI/CD (GHCR + SSH 배포)완료
Discord Webhook infra-discord 모듈완료
피드백 API + 서버 에러 알림완료
인프라 코드리뷰 실수 3건 수정완료
PR #13, #14 머지완료

BE 진행률 80~85% → 90~95%. 배포 인프라까지 갖춰졌다. 남은 건 실서버 배포 후 Gemini 연동 테스트 정도.
