## 자막에 재료 양이 없다

레시피 파싱의 핵심 문제를 발견했다. 기존 파이프라인은 yt-dlp로 YouTube 자막(ASR)을 추출해서 텍스트로 Gemini에 보내는 방식이었다. 근데 자막에 재료 양 정보가 없다. "소금 한 꼬집"은 말로 하니까 자막에 나온다. "돼지고기 300g"은 영상 화면에 텍스트 오버레이로만 뜬다. 자막만 읽으면 재료 양을 모르는 거다.

레시피 앱에서 재료 양을 모르면 쓸모가 없다.

## 비디오 직접 분석으로 전환

Gemini API가 YouTube URL을 `file_data.file_uri`로 직접 받는다. 영상을 통째로 분석한다. 음성 + 자막 + 화면 텍스트를 동시에 보되, 화면 텍스트를 우선한다.

```
Before: URL → yt-dlp(자막 추출) → 텍스트 → Gemini
After:  URL → Gemini(비디오 직접 분석)
```

자막 추출 단계가 사라졌다. `SubtitleExtractorPort` 인터페이스 삭제, `ParseWorker`에서 자막 추출 로직 제거. 코드가 단순해졌다.

### 비용과 속도

비디오 분석은 텍스트보다 오래 걸린다. WebClient timeout을 3분으로 늘렸다. 비용은 60초 Shorts 기준 약 15,500 토큰. 무료 tier로 하루 8시간 분량. 개인 프로젝트라 충분하다.

`gemini-2.0-flash-lite`가 deprecated 돼서 `gemini-2.5-flash-lite`로 업그레이드도 같이 했다. 새 사용자 기준으로 2.0 모델이 404를 뱉고 있었다.

### 프롬프트 개선

단위 변환 기준이 US cup(236.5ml)이었는데 한국 레시피와 안 맞다. 250ml(metric cup)로 수정. 자막 원본 단위를 우선 사용하고, 자막 내 변환 예시가 있으면 그 기준을 따르도록 지시. 가열 조리 단계에 불 세기도 필수로 넣게 했다.

비표준 단위(큰술, 스푼 등)가 영문으로 반환되는 경우도 있어서 한글 표기 규칙도 추가. 표준 단위계(g, kg, ml)는 영문, 비표준은 한글.

## yt-dlp 정리

자막 추출은 없앴지만 썸네일 추출은 남겨야 한다. 포트를 분리했다.

| Before | After |
|--------|-------|
| `SubtitleExtractorPort` | 삭제 |
| `YtDlpSubtitleExtractor` | `YtDlpThumbnailExtractor`로 교체 |
| `ThumbnailService` → `SubtitleExtractorPort` 의존 | `ThumbnailExtractorPort`로 변경 |

Docker에서 yt-dlp WARNING이 stdout에 섞여 썸네일 URL 파싱이 깨지는 버그도 있었다. `redirectErrorStream(true)` 때문. 출력에서 `http`로 시작하는 마지막 줄만 추출하도록 수정.

`ParseJobErrorCode.NO_SUBTITLE` enum은 DB 호환을 위해 삭제하지 않고 유지. 이미 저장된 에러 코드가 있을 수 있으니까.

## YouTube URL 정규화

같은 영상이라도 공유할 때마다 `si` 파라미터가 달라진다. 같은 사람이 같은 영상을 등록하면 다른 URL로 인식돼서 두 번 파싱. [분산 락](/posts/분산-락/)이나 캐시로 중복을 막고 있었는데, URL 자체가 달라버리니 무력화.

`UrlNormalizer` 유틸을 만들었다.

| 입력 | 출력 |
|------|------|
| `youtu.be/ID?si=abc` | `youtube.com/watch?v=ID` |
| `youtube.com/shorts/ID?feature=share` | `youtube.com/watch?v=ID` |
| `youtube.com/watch?v=ID&si=xyz` | `youtube.com/watch?v=ID` |
| `instagram.com/reel/CODE/?igsh=abc` | `instagram.com/reel/CODE/` |

`si`, `feature`, `igsh` 같은 추적 파라미터 제거. 정규화된 URL로 해시/저장/중복체크를 수행하니까 같은 영상은 한 번만 파싱.

유닛 테스트도 추가. 같은 영상의 다른 공유 파라미터가 동일 결과를 반환하는지, null/blank 엣지 케이스까지 검증.

## Claude Code 협업

Gemini 멀티모달 전환은 내가 판단. 자막에 재료 양이 없다는 문제를 발견하고, Gemini 문서에서 `file_data.file_uri`로 YouTube URL을 직접 보낼 수 있다는 걸 확인한 뒤 Claude에 지시. ObjectMapper로 request body를 조립하도록 하면서 수동 JSON escape도 같이 제거.

URL 정규화는 "같은 영상인데 si 파라미터 때문에 다른 URL로 인식된다, 정규화 유틸 만들어"라고 지시. `youtu.be`, `shorts`, `watch` 전부 통일하는 건 Claude가 알아서 처리. Instagram reel도 같이 넣어줬다.
