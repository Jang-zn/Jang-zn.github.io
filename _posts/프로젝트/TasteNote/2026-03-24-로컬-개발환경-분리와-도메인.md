## 배포용과 로컬이 한 파일에 있으면

[어제](/posts/Swagger-테스트-FE연결/) BE 정리를 끝내고, 오늘은 FE와 BE를 실제로 연결하려 했다. 근데 로컬에서 BE를 띄우는 것부터 꼬였다.

배포용 `docker-compose.yml` 하나에 prod/dev/local이 섞여 있었다. 로컬에서 개발하려면 불필요한 prod/dev 컨테이너까지 띄워야 하고, dev 컨테이너에 local 프로필을 씌우는 등 설정이 꼬였다. 한 서랍에 여름옷 겨울옷 다 넣어두고 반팔 하나 찾으려는 꼴이다.

## docker-compose 분리

`docker-compose.local.yml`을 새로 만들었다.

| 파일 | 용도 | 서비스 |
|------|------|--------|
| `docker-compose.yml` | 홈서버 배포 전용 | prod/dev API 4개 + MySQL + Redis 2개 + Nginx + 모니터링 |
| `docker-compose.local.yml` | 로컬 개발 전용 | local API 2개 + MySQL + Redis |

local 파일의 API 서비스는 `build:` 지시자로 소스에서 직접 이미지 빌드. 환경변수 기본값을 전부 넣어서 `.env` 파일 없이도 바로 실행 가능.

```bash
# 로컬 환경 한 줄로 기동
docker compose -f docker-compose.local.yml up -d --build
```

`application-local.yml`에도 환경변수 폴백 패턴을 적용해서 Docker 실행이든 bootRun이든 양쪽 다 호환.

## 포트 정리

기존에 포트가 꼬여 있었다.

| 환경 | API 포트 | Admin 포트 |
|------|---------|-----------|
| prod | 8080 | 8081 |
| dev | 8180 | 8181 |
| local | 8082 (신규) | 8083 (신규) |

MySQL은 3307로 호스트 포트를 매핑해서 bootRun으로 직접 실행할 때도 연결 가능하게. Redis는 6379 그대로.

## 토스 API mock 빈 분리

토스 파트너 API 인증을 로컬에서 우회하기 위해 `AuthController`에 dev bypass 코드가 박혀 있었다. 매직 스트링 `DEV_AUTH_CODE`, `@Value`로 프로필 체크, if-else 분기. 프로덕션 코드에 개발용 뒷문이 열려 있는 거다.

`LocalDevConfig`를 만들어서 `@Profile("local")`에서만 `TossPartnerApiClient`를 mock 빈(`@Primary`)으로 교체. 컨트롤러에서 bypass 코드 전부 삭제. 프로필과 무관하게 동일한 코드 경로.

## CORS

FE(Vite dev server, `localhost:5173`)에서 local BE(`localhost:8082`)로 요청 시 CORS preflight가 403. `WebConfig` CORS 허용 목록에 `localhost:5173` 추가.

## 도메인 연결 준비

[홈서버 배포](/posts/홈서버-배포-인프라/)에서 Cloudflare Tunnel을 설정해뒀다. 오늘 도메인을 사서 서브도메인을 연결할 준비를 했다. DNS 레코드, Nginx 리버스 프록시, HTTPS/SSL 인증서에 대해서는 [학습 포스트](/posts/도메인-DNS-Nginx-리버스-프록시/)에 정리했다.

핵심은 모든 서브도메인이 같은 Cloudflare Tunnel CNAME을 가리키고, Nginx가 Host 헤더로 분기하는 구조. `api.도메인` → `:8082`, `admin.도메인` → `:8084`, `grafana.도메인` → `:3000`.

## Claude Code 협업

docker-compose 분리는 "배포용/로컬용 분리해줘, local은 build 지시자로"라고 지시. `application-local.yml` 환경변수 폴백 패턴도 Claude가 잡아줬다.

토스 API mock 빈 분리는 내가 먼저 "컨트롤러에 dev bypass 코드가 있는 게 마음에 안 든다"고 했더니 `@Profile` + `@Primary` 조합을 제안. 깔끔하다.
