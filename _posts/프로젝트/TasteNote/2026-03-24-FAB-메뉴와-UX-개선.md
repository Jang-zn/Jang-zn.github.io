## 실제로 써보니까 불편한 것들

FE-BE를 연결하고 실제로 앱을 써보니까 UI에서 불편한 게 보이기 시작했다. 목 데이터로 개발할 때는 몰랐던 것들. 오늘 FE 쪽 UX 개선 커밋이 10개 넘게 나왔다.

## FAB 메뉴

레시피 상세페이지의 수정/새 버전 버튼이 HeroImage 안에 고정돼 있었다. 삭제 같은 새 액션을 추가하려면 HeroImage에 props를 계속 늘려야 한다. 확장성이 없다.

`ExpandableFAB` 컴포넌트를 만들었다. 우하단에 떠 있는 버튼. 누르면 위로 액션 버튼들이 펼쳐진다.

- 트리거 아이콘: ⋮(더보기). 열리면 ×
- 액션: 새 버전 → 수정 → 삭제 (사용 빈도순)
- 딤 오버레이, 외부 탭 닫힘, CSS keyframe 입장 애니메이션
- 아이콘 + 라벨 pill 형태. `minWidth: 120px`로 가로 통일
- `hexToRgba` 유틸로 액션 색상 자동 배경 생성

HeroImage에서 `onEdit`/`onNewVersion` props와 하단 버튼 영역 제거. 코치마크도 FAB 메뉴 안내 1개 스텝으로 통합 (기존 3스텝 → 2스텝).

## 삭제 기능

`useDeleteRecipe` 훅: `client.delete` + `removeQueries`/`invalidateQueries`. `DeleteConfirmSheet`: 기존 `BottomSheet` 재사용해서 삭제 확인 UI. MSW DELETE 핸들러 추가.

FAB에서 삭제 클릭 → `DeleteConfirmSheet` → mutation → 토스트 + 홈 이동.

## 이미지 에디터

수정/새버전 폼에서 사진을 바꿀 수가 없었다. `ImageEditor` 컴포넌트를 만들었다.

- 높이 180px. 현재 썸네일 표시, 없으면 카테고리 그라디언트 fallback
- 우하단 "사진 변경" 버튼 → hidden file input 트리거
- 업로드 중 로딩 오버레이
- 기존 `useUpdateRecipeImage` 훅 재사용

`RecipeEditPage`, `NewVersionPage` 스크롤 영역 최상단에 배치.

## 레시피 수정 페이지 보강

수정 폼에 타이틀/카테고리 편집 기능이 없었다. `useUpdateRecipeMeta` 훅을 만들어서 title + category를 함께 전송. 카테고리 칩 선택 UI도 추가.

저장 로직은 mutation 체이닝 방식으로. 메타데이터가 변경됐으면 `updateMeta` → `onSuccess` → `patchRecipe`. 변경 안 됐으면 바로 `patchRecipe`. 두 개를 동시에 쏘면 `invalidateQueries` 순서가 꼬인다. 자세한 건 [학습 포스트](/posts/레시피-폼-UX-개선-프론트엔드-실전-패턴/)에 정리.

## 카테고리 필터

TDS Tab 기본 스타일에서 선택 상태 시인성이 낮았다. 어떤 카테고리가 선택돼 있는지 한눈에 안 들어온다.

커스텀 pill 버튼으로 교체. 선택 상태에 카테고리별 `CATEGORY_COLORS` 배경 + 흰 텍스트. 미선택은 `C.bgSecond` 배경 + `C.gray500` 텍스트. `@toss/tds-mobile` Tab 컴포넌트 의존 제거.

chips 배열을 모듈 상수(`CHIPS`)로 호이스팅해서 렌더마다 재생성하지 않게. 우측 끝에 `mask-image` 페이드 효과로 "더 있어요" 힌트.

## 이미지 가독성

HeroImage와 RecipeCard의 이미지 위 텍스트가 잘 안 보였다. 그라디언트 딤이 약했다. 하단 불투명도를 0.82까지 올리고, 제목에 `textShadow` 추가. 리스트는 "버전 · 날짜", 상세는 "날짜 · 버전"으로 순서가 달랐는데 통일.

재료 카드에서 필수 재료가 `C.primary`(붉은 주황) 텍스트로 표시돼 있었다. 경고처럼 보인다. 텍스트 색상을 통일하고, 우상단에 "필수"(파란 뱃지) / "선택"(회색 뱃지)로 변경.

## 에디터 컴팩트화

재료/단계 에디터의 항목이 많을 때 한눈에 안 들어왔다. padding, gap, fontSize를 전체적으로 줄임. 항목 추가 시 자동 스크롤 + 하이라이트 애니메이션(600ms). StepEditor 드래그 핸들/순서 뱃지 수직 정렬 보정.

`useScrollHighlight` 훅으로 IngredientEditor와 StepEditor에 중복된 scroll+highlight 로직을 추출. `setTimeout` cleanup도 추가해서 메모리 누수 제거.

## 원본 영상 링크

레시피의 원본 영상 URL이 있어도 접근할 방법이 없었다. HeroImage 우상단에 "원본 영상" 버튼 추가. `originUrl` 있을 때만 표시. `window.open`에 `noopener,noreferrer` 플래그. 모바일에서는 OS가 YouTube 앱으로 자동 라우팅.

## FE 정렬 제거

[BE에서 steps를 order 순으로 정렬](/posts/SSE-실시간-알림과-API-보강/)해서 내려주기로 했다. FE에서 `[...steps].sort()` 제거. 서버가 정렬을 보장하면 FE에서 중복 정렬할 필요 없다. 배달 앱에서 음식이 순서대로 포장되어 온다고 약속했는데 집에서 또 정렬하면 불필요한 작업이다.

## Claude Code 협업

FAB, 삭제, 이미지 에디터는 각각 "이런 컴포넌트 만들어"로 지시. 특히 FAB는 "ExpandableFAB 범용 컴포넌트로, 액션은 props로 받게"라고 했더니 확장 가능한 구조로 잘 만들어줬다.

카테고리 필터 pill 교체는 "TDS Tab 시인성 낮다, 커스텀 pill로 바꿔"로 지시. mask-image 페이드 효과는 Claude가 알아서 추가. chips 호이스팅도.

에디터 컴팩트화와 `useScrollHighlight` 추출은 "두 에디터에 중복 로직이 있다, 훅으로 빼고 setTimeout cleanup도 추가해"로 지시.
