[TasteNote] 코치마크 튜토리얼 구현

By 코드대장간
Posted Mar 20 Updated Mar 20 11 min read

코치마크

신규 사용자가 앱에 처음 들어왔을 때 “이 버튼이 뭐고, 저기서 뭘 할 수 있는지” 알려주는 튜토리얼 오버레이를 만들었다. 홈 4스텝, 상세 3스텝, 등록 2스텝. 총 9스텝.

어제 camelCase 마이그레이션 끝내고 바로 들어간 작업이다.

구조

6개 커밋으로 나눠서 진행했다.

  1. localStorage 팩토리 — 공용 유틸 추가
  2. 코치마크 피처 — 스토어, UI, 훅 전체 구현
  3. 페이지 연결 — 기존 컴포넌트에 data-coach 속성 부여 + 오버레이 마운트
  4. 타겟 요소 미발견 대응 — MutationObserver 복구 + 폴백 UI
  5. ResizeObserver 가드 + 툴팁 높이 실측 — 호환성 및 위치 정확도 개선
  6. 스타일 상수 추출 — Tooltip/FallbackOverlay 중복 인라인 스타일 정리

localStorage 팩토리

코치마크 완료 여부를 저장하려면 localStorage가 필요하다. 근데 기존에 배너 dismiss에서도 localStorage를 직접 쓰고 있었고, 에러 핸들링(시크릿 모드, 용량 초과)이 빠져있었다. 공용 유틸로 뽑았다.

createFlag(prefix) — 영구 플래그. 코치마크 완료 저장용. createDailyFlag(prefix) — 당일 한정 플래그. 배너 dismiss용.

내부에서 safeGet/safeSet으로 try-catch를 통합해서, 시크릿 모드나 용량 초과에서도 앱이 안 죽는다. 자세한 설계 배경은 학습 포스트에 정리.

코치마크 피처

FSD(Feature-Sliced Design) 구조로 features/coach-mark/ 밑에 전부 모았다.

스포트라이트 — SVG Mask

화면 전체를 반투명으로 덮고, 타겟 요소만 투명하게 뚫는다. SVG Mask 학습 포스트에 원리를 정리했다.

useId()로 mask ID를 생성해서 같은 페이지에 여러 SVG가 있어도 충돌하지 않게 했다. 어두운 영역 클릭 시 skip 처리, 타겟 영역은 pointerEvents: 'none'으로 클릭 투과.

위치 추적 — ResizeObserver + scroll

타겟 요소의 위치가 바뀌면 스포트라이트도 따라가야 한다. ResizeObserver로 크기 변화를 잡고, scroll 이벤트로 스크롤 변화를 잡는다. 이전 rect와 비교해서 실제로 변했을 때만 스토어를 업데이트한다. 불필요한 리렌더 방지.

툴팁 위치 계산 — flip 지원

calcTooltipPosition에서 타겟 아래에 공간이 부족하면 위로 뒤집는다(flip). 좌우도 뷰포트 경계를 넘지 않게 클램핑. 화살표 위치도 타겟 중심에 맞춘다.

Zustand 스토어 — 상태 머신

activeTutorial이 null이면 비활성, 값이 있으면 활성. start()에서 coachMarkStorage.isDone() 체크해서 이미 완료한 튜토리얼은 시작하지 않는다. complete()에서 markDone() 호출해서 영구 저장. skip()은 저장 안 함 — 다음에 다시 나온다.

구현 후 버그 수정 3건

만들고 나서 돌려보니 문제가 나왔다.

타겟 요소가 없으면 먹통

data-coach 타겟 요소가 조건부 렌더링이나 느린 로딩으로 아직 DOM에 없으면, useCoachMark가 바로 return해버린다. targetRect가 null인 채로 튜토리얼이 “보이지 않는 상태”에서 멈추고, 사용자가 skip할 방법이 없다. 앱이 먹통처럼 보이는 버그.

두 가지로 해결했다. 첫째, 요소가 없으면 MutationObserver로 DOM 변화를 감시하다가 나타나면 관찰을 시작한다. 3초 기다려도 안 나타나면 nextStep()으로 자동 스킵. 둘째, targetRect가 null일 때 중앙 정렬된 폴백 UI(메시지 + 건너뛰기 버튼)를 보여준다. 배경 클릭으로도 dismiss 가능.

툴팁 위치가 부정확

calcTooltipPosition이 툴팁 높이를 100px 고정값으로 계산하고 있었다. 메시지가 길면 실제 높이가 달라져서 flip 판단이 틀어진다. 툴팁이 뷰포트 밖으로 나가는 경우가 생겼다.

callback ref로 DOM 렌더 후 실제 높이를 측정하고, 그 값을 calcTooltipPosition에 넘기도록 수정. 스텝이 바뀔 때마다 재측정한다.

ResizeObserver 미지원 환경

구형 Safari나 WebView, 테스트 환경에서 ResizeObserver가 없으면 런타임 에러가 난다. typeof ResizeObserver 체크를 넣고, 미지원 시 window resize 이벤트로 폴백하게 했다.

스타일 중복 정리

Tooltip과 FallbackOverlay에서 카드, 메시지, 버튼 스타일이 인라인으로 중복되어 있었다. styles.ts에 공유 상수로 추출. 순수 리팩토링이라 렌더링 결과는 동일.

Claude Code 협업

“코치마크 기능 만들어줘”라고 했을 때 Claude가 SVG mask, ResizeObserver, Zustand, localStorage 팩토리까지 한번에 구현을 내놨다. 전체 구조 설계는 Claude가 했고, 내가 한 건 방향 잡기랑 세부 조정이다.

useId()로 mask ID 충돌 방지한 건 Claude 판단. prevRectRef로 불필요한 업데이트를 걸러내는 것도. 코치마크처럼 “전체 흐름이 정해진” 피처는 Claude가 빠르게 잘 만든다.

다만 페이지 연결은 내가 직접 어떤 요소에 data-coach를 붙일지 정했다. “홈에서 뭘 가르쳐줘야 하는지”는 UX 판단이니까.

버그 수정 3건도 Claude가 처리했다. “타겟 없으면 먹통인데?”라고 짚으면 MutationObserver 폴백을 구현하고, “툴팁 높이 고정값이면 긴 메시지에서 터지는데?”라고 하면 callback ref로 실측 기반으로 바꾸는 식. 문제를 알려주면 해결 방법은 빠르게 잡는다.

배너 dismiss에 기존에 쓰던 인라인 localStorage 코드를 createDailyFlag로 리팩토링한 것도 이번에 같이 처리. 기존 useDismissedToday 훅에서 try-catch + 날짜 비교 로직이 빠지면서 17줄 → 4줄로 줄었다.
