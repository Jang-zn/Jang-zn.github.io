## BE가 SSE를 쏘면 FE가 받아야 한다

[같은 날 BE에서](/posts/SSE-실시간-알림과-API-보강/) SSE 엔드포인트를 만들었다. FE에서 구독해서 사용자에게 보여줘야 한다.

## SSE 구독 훅

`useParseJobSSE` 훅. fetch 기반 SSE 구독. EventSource를 안 쓴 이유는 JWT 인증 헤더를 보내야 해서. fetch로 `text/event-stream`을 직접 읽는다. 120초 타임아웃, 언마운트 시 AbortController로 안전 종료.

MSW 핸들러도 만들었다. 3초 후 DONE 이벤트 전송, stream cancel 시 타이머 정리.

## 등록 페이지 → 앱 레벨 스낵바

처음에는 등록 페이지에서 SSE 완료를 대기하게 했다. 제출 후 `ParseProgress` 컴포넌트로 처리 상태 표시. DONE이면 토스트 + 레시피 상세 이동, FAILED면 에러 메시지 + 재시도 버튼.

근데 문제가 있었다. 완료 후 홈으로 돌아가면 새 레시피가 목록에 안 나타난다. `invalidateQueries` 누락. 등록 페이지에서 SSE를 구독하고 있으니, 홈으로 이동하면 구독이 끊긴다. 레시피 목록 캐시 갱신 타이밍을 놓치는 거다.

앱 레벨로 올렸다.

| 변경 전 | 변경 후 |
|---------|---------|
| 등록 페이지에서 SSE 구독 | 앱 루트에서 전역 구독 |
| 완료까지 등록 페이지에 머무름 | 제출 → 홈 이동 → 스낵바로 결과 확인 |
| invalidateQueries 타이밍 문제 | 어느 화면에서든 캐시 갱신 |

`parseJobStore`에 활성 job ID를 전역으로 관리. `ParseJobSnackbar`를 앱 루트에 마운트.

- 처리중: 스피너 + "AI가 레시피를 분석하고 있어요..."
- 성공: 녹색 + "레시피가 등록되었어요!" + `invalidateQueries`
- 실패: 빨간색 + 에러 메시지
- 3초 후 자동 dismiss

등록 페이지는 제출만 담당. URL 입력 → 제출 → store에 jobId 저장 → 홈 이동.

## 인터스티셜 광고

레시피 등록 수익화. 1일 1회 무료, 이후 전면광고 시청 후 등록.

### 카운터

`createDailyCounter`를 만들었다. localStorage에 `{ date, count }` JSON 저장. 날짜가 바뀌면 자동 리셋. [코치마크](/posts/코치마크-튜토리얼-구현/)에서 만든 `createFlag`/`createDailyFlag`와 같은 패밀리.

### AdAdapter 확장

[이전에](/posts/피드백-기능과-메인-컬러-변경/) 배너 광고용 AdAdapter 패턴을 만들어뒀다. 여기에 `loadInterstitial`/`showInterstitial` 메서드를 추가.

`WebAdAdapter`에서는 `window.confirm` 기반 mock. `TossAdAdapter`에서는 `TossAds.loadFullScreenAd`/`showFullScreenAd`를 콜백 → Promise 래핑. 5초 타임아웃 + settled 가드로 브릿지 무응답 시에도 안전하게 resolve.

### 게이팅 흐름

```
등록 버튼 클릭
  → createDailyCounter로 당일 횟수 확인
    → 0회: 광고 없이 통과
    → 1회 이상: showInterstitial
      → 성공/실패 상관없이 등록 진행
```

광고 실패 시에도 등록 진행한다. reject 없이 resolve. 광고 때문에 서비스가 막히면 본말전도다. 자판기에 동전을 넣었는데 광고가 안 나온다고 음료수가 안 나오면 안 된다.

`RegisterPage.handleSubmit`을 async로 변경, `await gateInterstitial()` 후 `submitJob.mutate()` 실행.

## Claude Code 협업

SSE 훅은 "fetch 기반으로 SSE 구독하는 훅 만들어, JWT 헤더 포함"으로 지시. EventSource 대신 fetch를 쓰는 판단은 내가 했다.

등록 페이지 → 앱 레벨 스낵바 리팩토링은 직접 사용하면서 발견한 문제. "SSE 구독을 앱 루트로 옮기고, 등록 페이지는 제출만 담당하게 바꿔"라고 지시.

인터스티셜 광고는 "1일 1회 무료, 이후 전면광고, 실패해도 등록 진행"이라는 요구사항을 Claude에 전달. `createDailyCounter`, `gateInterstitial` 구조를 잘 잡아줬다. 5초 타임아웃이나 settled 가드 같은 안전장치도 Claude가 제안.
