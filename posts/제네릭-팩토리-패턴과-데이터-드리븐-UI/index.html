By 코드대장간
Posted Mar 25 Updated Mar 26 3 min read ⭐6

같은 코드가 3번 복사돼 있었다

앱인토스 Admin에서 데이터 내보내기 기능이 3개 있었다. 사용자 목록 내보내기, 레시피 목록 내보내기, 결제 목록 내보내기. 각각 별도 API 함수와 별도 hook이 있었는데, 로직이 거의 같았다.

// exportUsers.ts
const exportUsers = async () => { fetch('/admin/v1/users/export'); ... }

// exportRecipes.ts
const exportRecipes = async () => { fetch('/admin/v1/recipes/export'); ... }

// exportPayments.ts
const exportPayments = async () => { fetch('/admin/v1/payments/export'); ... }

URL만 다르고 나머지가 동일하다. 복사-붙여넣기의 산물.

제네릭 팩토리로 통합

type ExportKey = 'users' | 'recipes' | 'payments';

const EXPORT_ENDPOINTS: Record<ExportKey, string> = {
  users: '/admin/v1/users/export',
  recipes: '/admin/v1/recipes/export',
  payments: '/admin/v1/payments/export',
};

async function exportData(key: ExportKey) {
  const response = await client.get(EXPORT_ENDPOINTS[key], { responseType: 'blob' });
  // 공통 다운로드 로직
}

3개 함수가 1개로 줄었다. 새 내보내기가 추가되면 EXPORT_ENDPOINTS에 한 줄 추가하면 된다. 자판기에 음료를 추가할 때 새 자판기를 만드는 게 아니라 버튼 하나를 추가하는 거다.

데이터 드리븐 설정

행동(코드)과 설정(데이터)을 분리하는 패턴이다.

1
2
3
4
5

const EXPORT_CONFIG: Record<ExportKey, { label: string; filename: string }> = {
  users: { label: '사용자 목록', filename: 'users.csv' },
  recipes: { label: '레시피 목록', filename: 'recipes.csv' },
  payments: { label: '결제 목록', filename: 'payments.csv' },
};

UI에서 Object.entries(EXPORT_CONFIG).map()으로 버튼을 동적 생성. 설정 객체가 UI를 결정한다.

로딩 상태 분리

기존에는 isLoading: boolean 하나로 3개 내보내기의 로딩을 관리했다. 사용자 내보내기 중에 레시피 내보내기 버튼도 비활성화되는 문제.

1
2
3
4
5

const [loadingMap, setLoadingMap] = useState<Record<ExportKey, boolean>>({
  users: false,
  recipes: false,
  payments: false,
});

Record<Key, boolean>로 각각 독립적인 로딩 상태를 관리. 화장실 3칸에 “사용 중” 표시가 각각 있는 거다. 한 칸이 사용 중이라고 나머지도 못 쓰면 안 된다.

Spring의 Strategy 패턴과 비교

BE의 Spring에서도 같은 패턴을 쓴다.

1
2
3
4
5
6

Map<String, ExportHandler> handlers = Map.of(
  "users", userExportHandler,
  "recipes", recipeExportHandler
);

handlers.get(key).export();

FE의 Record<Key, Config> + 제네릭 함수와 BE의 Map<String, Handler> + Strategy 패턴은 같은 원리다. “키로 행동을 선택한다.”

프로젝트 적용

useExport 훅도 제네릭으로 리팩터링. exportData(key) 호출 + loadingMap[key] 상태 관리를 하나의 훅으로 통합. 3개 파일(200줄) → 1개 파일(60줄).
