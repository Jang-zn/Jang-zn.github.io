TasteNote FE-BE 연결과 UX 개선을 하면서 프론트엔드 실전 패턴을 여럿 만났다. 그중 반복해서 쓸 만한 것들을 정리한다.

Hidden File Input

브라우저의 기본 파일 선택 UI는 못생겼다. 숨겨놓고 예쁜 버튼으로 대신 트리거한다. 카페의 직원 전용 출입구. 손님 눈에는 안 보이지만 벨을 누르면 문이 열린다.

1
2
3
4
5
6

const inputRef = useRef<HTMLInputElement>(null);

<input ref={inputRef} type="file" accept="image/*"
  onChange={handleFileChange} style={{ display: 'none' }} />

<button onClick={() => inputRef.current?.click()}>사진 변경</button>

주의할 점: e.target.value = ''를 안 하면 같은 파일을 다시 선택할 때 onChange가 안 된다. 브라우저가 “값이 안 바뀌었으니 이벤트 안 쏠게”라고 판단한다.

이미지 에디터 컴포넌트에서 이 패턴을 적용했다.

CSS mask-image

카테고리 칩이 오른쪽 끝에서 갑자기 잘리면 “여기가 끝인가?” 혼란을 준다. mask-image로 자연스럽게 페이드시키면 “더 있어요” 힌트가 된다. 카페 유리창에 안개 스프레이를 뿌려서 바깥이 서서히 흐려지게 만드는 거다.

1
2

maskImage: 'linear-gradient(to right, black calc(100% - 40px), transparent)',
WebkitMaskImage: 'linear-gradient(to right, black calc(100% - 40px), transparent)',

Safari/iOS WebView에서 maskImage를 접두사 없이 지원하지 않는 경우가 있다. 토스 앱은 WebView 기반이니까 WebkitMaskImage도 같이 써야 한다.

useEffect + setTimeout 정리

사무실 에어컨(setTimeout)을 틀어놨으면 퇴근(언마운트)할 때 꺼야 한다.

// Bad: cleanup 없음. 컴포넌트 사라져도 setTimeout 실행됨
useEffect(() => {
  setTimeout(() => { el.style.background = 'transparent'; }, 600);
}, [items]);

// Good: clearTimeout으로 정리
useEffect(() => {
  const timerId = setTimeout(() => { ... }, 600);
  return () => clearTimeout(timerId);
}, [items]);

에디터에서 항목 추가 시 하이라이트 애니메이션에 이 패턴을 적용. useScrollHighlight 훅으로 추출하면서 cleanup도 같이 넣었다.

커스텀 훅 추출 기준

IngredientEditor와 StepEditor에 동일한 scroll+highlight 코드가 복붙돼 있었다. 김치찌개 레시피를 A 노트와 B 노트에 똑같이 적어놓은 거다. “두부 넣어야지”라고 수정할 때 둘 다 고쳐야 한다. 하나를 깜빡하면 버그.

useScrollHighlight 훅으로 추출. 사용하는 쪽은 한 줄이면 끝.

1

const { lastAddedKey, lastRowRef } = useScrollHighlight(ingredients);

추출 기준은 단순하다.

Mutation 체이닝

레시피 수정에서 메타데이터(제목/카테고리)와 버전(재료/단계) 두 개의 mutation을 보내야 한다. 동시에 쏘면 invalidateQueries 순서가 꼬인다. 택배를 보내고 나서 영수증을 출력해야 하는데, 둘을 동시에 하면 택배 도착 전에 영수증이 나온다.

1
2

const handleSave = () => {
  if (metaChanged) {
    updateMeta.mutate(
      { title, category },
      { onSuccess: saveVersion },  // 완료 후에 다음 단계
    );
  } else {
    saveVersion();
  }
};

앞쪽 mutation에서 invalidateQueries를 제거하고, 마지막 mutation에서만 invalidate. 같은 queryKey를 두 번 invalidate하면 같은 API를 2번 호출한다.

PUT vs PATCH

서버에 PATCH 엔드포인트가 없어서 버전 수정 시 500이 나왔다. PUT만 지원.

  • PUT = 이사: 짐을 다 빼고 새 짐을 통째로 넣는다. 안 보낸 필드는 null
  • PATCH = 가구 배치: 기존은 유지하고 소파만 바꾼다. 안 보낸 필드는 그대로

PUT에 category를 안 보내면 400 에러(“category: 널이어서는 안됩니다”). 전체 필드를 보내야 한다.

MSW mock도 같이 바꿔야 한다. API는 PUT인데 mock이 PATCH면 로컬에서 404.

서버 정렬 신뢰

BE에서 steps를 order 순으로 정렬해서 보내기로 했다. FE에서 .sort() 제거. 배달 앱에서 음식이 순서대로 포장돼 온다고 약속했으면 집에서 또 정렬할 필요 없다.

단, 서버 계약을 신뢰하고 FE 정렬을 제거하려면 커뮤니케이션이 전제다. 커밋 메시지에 근거를 남기거나 API 문서에 명시해야 한다. 구두 약속만으로는 부족하다.

모듈 상수 호이스팅

카테고리 필터의 chips 배열이 컴포넌트 안에 있어서 렌더마다 재생성됐다. 저녁에 파스타를 만들 때마다 마트에 가서 재료를 사오는 거다. 냉장고에 미리 사놓으면 안 가도 된다.

// 컴포넌트 밖에서 한 번만
const CHIPS = [
  { label: '전체', color: C.primary },
  ...CATEGORY_DISPLAY_ORDER.map(...)

props/state에 의존하지 않는 값은 모듈 상수. 의존하지만 자주 안 바뀌면 useMemo. 매번 바뀌면 인라인.

window.open 보안

원본 영상 링크를 외부에서 여는데, noopener,noreferrer 플래그가 필요하다. 없으면 새 탭에서 window.opener.location으로 원래 탭의 URL을 바꿀 수 있다. Reverse Tabnabbing. 내 가게에서 다른 가게로 손님을 안내했는데, 뒷문을 열어놔서 역침투 당하는 거다.

1

window.open(url, '_blank', 'noopener,noreferrer');

한 줄이면 막는다.
