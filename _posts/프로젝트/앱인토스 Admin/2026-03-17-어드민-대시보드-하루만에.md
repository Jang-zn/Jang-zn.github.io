## 세 번째

[BE](/posts/도메인-대수술과-어드민-API/), [TasteNote](/posts/어드민-연동-준비/) 다음. 어드민 대시보드 FE 차례.

목표: **앱인토스 플랫폼 전체를 관리하는 어드민**. 사용자, 콘텐츠, 공지사항, 배너, 피처 플래그, 결제, 모니터링. 중요한 건 어드민을 코어 API에 붙이지 않는 것. **관리하는 앱서버**라고 생각해야 한다. BE에서도 `admin-api`를 별도 모듈로 분리한 이유가 이거다.

## 하루에 21커밋 — Claude Code 전력 가동

진짜 하루 만에 전체 뼈대를 끝냈다. **21개 커밋**. Claude Code가 없었으면 절대 불가능했을 속도.

작업 방식이 좀 독특했다. [BE에서 만든 1,200줄 기획서](/posts/도메인-대수술과-어드민-API/)를 Claude한테 통째로 던지고 "이 스펙대로 어드민 FE 만들어줘"라고 했다. Claude가 프로젝트 구조부터 컴포넌트까지 쭉 만들어내는데, 처음에 Claude가 Write 하려는 걸 한번 거부했다. 내가 원하는 방향이 아니었거든. "이건 아닌데, 이런 식으로 해줘" 하고 다시 시키니까 방향을 잘 잡았다.

특히 `.pr_agent.toml`(Qodo Merge 설정) 만들 때가 좀 웃겼는데, 처음에 영어로 설정을 만들어서 "한국어로 해"라고 고치고, 또 언어 설정이 안 맞아서 `.codiumai.toml` 만들었다가 삭제하고... 설정 파일 하나에 커밋이 3개나 들어갔다.

## 기술 스택

```
React 19 + TypeScript (strict)
Vite
shadcn/ui + Carbon Design System 토큰
Zustand (상태 저장)
TanStack Query
React Router v7
Axios (JWT interceptor)
Recharts
```

### shadcn/ui + Carbon Design System

여기서 고민한 게 디자인 시스템이다. 사용자용 앱(TasteNote)은 토스 TDS를 따라야 하는데, 어드민은 토스 앱이 아니니까 다른 디자인 시스템을 써도 된다. **Carbon Design System v11**을 선택했다. IBM이 만든 거라 정보 밀도 높은 어드민 UI에 적합하다. shadcn/ui는 코드를 직접 갖고 있으니까 커스텀이 자유롭고, 장애인 접근성도 기본 지원된다. Carbon 컬러 토큰을 입혀서 톤 통일.

## Figma MCP — 디자이너 없이 디자인하기

디자이너가 없다. 근데 어드민이라도 너무 못생기면 안 되잖아. 여기서 **Figma MCP**가 빛을 발했다. Claude Code에 Figma MCP 도구가 연결되어 있어서, Figma 파일을 직접 읽으면서 컴포넌트 구조를 참조할 수 있었다.

"이 Figma 디자인 보고 비슷하게 만들어줘"라고 하면 레이아웃이랑 간격, 컬러 토큰을 참조해서 컴포넌트를 만들어줬다. 물론 100% 일치하진 않지만, 참고 자료가 있으니까 혼자 맨땅에서 시작하는 것보다 훨씬 나았다.

## 19개 공통 컴포넌트 + 4개 차트

재사용 가능한 공통 컴포넌트를 19개 만들었다:

- **레이아웃**: Sidebar, Header, PageContainer, ContentCard
- **데이터 표시**: DataTable, StatusBadge, StatCard, EmptyState
- **입력**: SearchInput, FilterBar, DateRangePicker
- **피드백**: ConfirmDialog, Toast, LoadingSpinner
- **네비게이션**: Breadcrumb, TabNav, Pagination

차트는 Recharts로 4종류:
- 일별 활성 사용자 (LineChart)
- 카테고리별 레시피 분포 (BarChart)
- 사용자 구성 (PieChart)
- 매출 추이 (AreaChart)

## 상태 관리 전략

**Zustand**는 UI 상태(사이드바 열림/닫힘, 테마, 필터 상태)에만 쓰고, **TanStack Query**가 서버 상태 전담. 이건 TasteNote에서 쓰던 패턴 그대로.

```typescript
// Zustand - UI 상태
const useSidebarStore = create(persist(...))
const useAuthStore = create(persist(...))

// TanStack Query - 서버 상태
const { data: users } = useQuery({ queryKey: ['users'], ... })
const { data: recipes } = useQuery({ queryKey: ['recipes'], ... })
```

## Qodo Merge 설정

PR 리뷰 자동화를 위해 `.pr_agent.toml`도 세팅했다. 한국어로 리뷰 코멘트 작성, TypeScript strict 모드 체크, React 성능 최적화 확인, XSS 방지 체크 등.

## 페이지 구성

사이드바 기반 레이아웃에 다음 페이지들:

- **대시보드**: 핵심 지표 카드 4개 + 차트 4개
- **사용자 관리**: 목록/상세/수정
- **레시피 관리**: 목록/상세, 신고 처리
- **운영**: 공지사항, 배너, 팝업 CRUD
- **설정**: 피처 플래그, 앱 버전, 시스템 설정
- **감사 로그**: 어드민 활동 이력 조회
