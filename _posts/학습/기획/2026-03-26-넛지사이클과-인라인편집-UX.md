## 요리하다가 자연스럽게 레시피를 개선하게

TasteNote의 핵심은 "레시피 버전 관리"다. 요리할 때마다 조금씩 바꿔서 나만의 레시피를 완성하는 것. 문제는 [진입점 분석](/posts/진입점-통합과-선택피로/)에서 발견했듯이, "새 버전" 버튼을 직접 누르는 사용자가 드물다는 것.

그게 왜 그런지를 파고들다 보니, 문제가 버튼 위치가 아니었다. 요리 모드 자체의 구조에 있었다.

## 원래 설계의 가정

요리 모드를 처음 설계할 때 가정이 있었다. "요리 중에는 레시피를 보기만 하면 된다. 수정은 요리 전후에 한다."

그래서 요리 모드는 읽기 전용이었다. 재료를 체크하고 단계를 따라가는 것만 가능했다. 레시피를 바꾸고 싶으면 요리 모드를 나가서 수정 페이지로 가야 했다.

이 가정이 틀렸다. 실제로 쓰다 보면 요리 중에 "소금을 좀 줄여봤는데 이게 더 나은데, 기록해야 하는데..."가 생긴다. 요리 모드를 나가야 하는 순간 흐름이 끊긴다.

## 첫 번째 시도: 임시 메모 추가

흐름이 끊기는 문제를 해결하려고 처음엔 "임시 메모" 기능만 추가하려 했다. 요리 중에 텍스트 한 줄 써놓을 수 있게.

근데 이렇게 하면 요리 완료 후 그 메모를 수동으로 수정 페이지에 옮겨야 한다. 같은 일을 두 번 하는 구조다. 레시피를 바꾸겠다는 의도가 생긴 시점에서 기록하고, 나중에 또 그걸 반영하는 것.

## 인라인 편집으로 전환

그래서 아예 방향을 바꿨다. 메모를 따로 받지 말고, 요리 중에 바로 수정하게.

재료를 탭하면 상태가 순환한다.

```
사용함(USED) → 조정함(ADJUSTED) → 건너뜀(SKIPPED) → 사용함
```

한 번의 탭으로 상태를 변경. Fitts's Law — 큰 타겟(56px 체크박스)을 가까이(재료 바로 옆) 배치하면 조작이 빠르다. 요리 중에 정교한 터치를 할 수 없다.

각 상태에 액션 뱃지가 표시된다. ADJUSTED는 수량 수정 입력이 나타나고, SKIPPED는 취소선.

단계에는 메모 토글 버튼. "다음에는 3분 대신 2분만 볶기" 같은 메모를 단계 바로 옆에 남긴다. 단계 추가/삭제도 가능.

## 넛지 사이클

인라인 편집이 생기니까 완료 시 흐름을 설계할 수 있었다.

```
요리 시작 → 재료 체크 → 인라인 편집 → 요리 완료
  → 변경 없음: "수고했어요" → 끝
  → 변경 있음: "새 버전으로 저장할까요?" → 넛지
```

요리 중에 변경이 발생했으면 완료 시 자동으로 넛지가 뜬다. "새 버전" 버튼을 찾아 누르지 않아도 된다. Path A(변경 없음)는 최소 단계로 종료. Path B(변경 있음)는 넛지로 유도.

## 완료 시 분기

`CookingCompleteSheet`가 변경 감지를 한다.

처음엔 `ingredientActions`만 체크했다. 재료 상태가 바뀌었으면 변경이 있는 거라고. 근데 단계 메모도 변경으로 봐야 한다는 걸 나중에 추가했다. 메모를 썼는데 넛지가 안 뜨면 이상하다.

```typescript
const hasModifications = useMemo(() => {
  return Object.values(ingredientActions).some(a => a !== 'USED')
    || stepMemos.some(m => m.length > 0)
    || addedSteps.length > 0
    || deletedSteps.length > 0;
}, [ingredientActions, stepMemos, addedSteps, deletedSteps]);
```

- 변경 없음 → "수고했어요! 요리 완료" 버튼만
- 변경 있음 → "새 버전으로 저장" 버튼이 추가로 표시

"새 버전 저장"을 누르면 [location.state](/posts/location-state와-페이지간-데이터-전달/)로 변경 데이터를 넘기고 NewVersionPage로 이동. 폼이 요리 중 변경사항으로 프리필된다.

## 프리필 전략

NewVersionPage는 두 가지 진입 경로가 있다.

| 진입 | 프리필 | 출처 |
|------|--------|------|
| 요리 모드에서 | 변경된 재료/단계/메모 | location.state |
| 직접 접근 | 현재 버전 복사 | API 조회 |

`fromCooking` 플래그로 분기. 요리 모드에서 왔으면 `cookingEdits`로 폼을 채우고, 아니면 기존 레시피를 복사.

## ReviewSheet 삭제

원래 요리 완료 시 별점 + 메모를 입력하는 `CookingReviewSheet`가 있었다. 처음엔 둘 다 두려고 했다. 인라인 편집도 있고, 완료 후 메모도 받고.

근데 이게 "같은 질문을 두 번"이 되는 거였다. 요리 중에 이미 메모를 했는데 완료 후 또 메모를 받는다. 사용자 입장에서 짜증난다. 인라인 편집이 생기면서 `ReviewSheet`는 의미가 없어졌다. 그냥 삭제했다.
