By 코드대장간
Posted Mar 24 Updated Mar 26 4 min read ⭐3

도메인은 주소표일 뿐이다

TasteNote 홈서버에 도메인을 연결하면서 정리한 내용이다. 도메인은 그 자체로 아무것도 보여주지 않는다. IP 주소에 이름을 붙여주는 것일 뿐.

브라우저가 api.도메인.com에 접속하면:

  1. DNS가 IP 주소를 알려준다
  2. 브라우저가 그 IP의 80포트로 HTTP 요청을 보낸다 (Host 헤더에 도메인 포함)
  3. 서버에서 80포트를 듣고 있는 프로그램이 응답한다

서버에 아무것도 안 띄워놨으면 ERR_CONNECTION_REFUSED. Nginx가 떠 있으면 Nginx가 응답. 도메인은 주소표이고, 응답은 서버에서 돌고 있는 프로그램이 한다.

Nginx 리버스 프록시

서브도메인이 여러 개인데 전부 같은 서버를 가리킨다. api., admin., grafana. 전부 같은 IP. 그럼 어떻게 구분하는가?

Nginx가 안내데스크 역할을 한다. 모든 요청이 80포트로 들어오면, HTTP Host 헤더를 보고 각각 다른 Docker 컨테이너 포트로 프록시.

server {
    listen 80;

    location / { proxy_pass http://localhost:8082; }

server {
    listen 80;
    server_name admin.도메인.com;
    location / { proxy_pass http://localhost:8084; }
}

server {
    listen 80;
    server_name grafana.도메인.com;
    location / { proxy_pass http://localhost:3000; }
}

전부 80포트로 들어오지만, 도메인별로 다른 서비스에 연결된다. 호텔 프론트에서 객실 번호를 알려주는 거다. 모든 손님이 로비로 들어오지만, 각자 다른 방으로 안내받는다.

Cloudflare Tunnel을 포함한 전체 흐름

홈서버는 공유기 뒤에 있어서 외부에서 직접 접근할 수 없다. Cloudflare Tunnel이 이 문제를 해결한다.

1

브라우저 → HTTPS → Cloudflare → Tunnel → cloudflared → HTTP → Nginx → Spring Boot
구간역할
CloudflareDNS + HTTPS 인증서 + CDN
cloudflared서버→Cloudflare 아웃바운드 연결 (포트 개방 불필요)
Nginx도메인 기반 라우팅
Docker 컨테이너실제 서비스

cloudflared가 서버에서 Cloudflare로 먼저 연결을 맺는 아웃바운드 터널이라서, 공유기에서 80/443 포트를 열 필요가 없다. 보안상 큰 장점.

환경 분리

docker-compose에서 포트로 환경을 분리해뒀다. 서브도메인으로 환경을 구분.

서브도메인컨테이너포트
api.도메인prod API8082
dev-api.도메인dev API8083
admin.도메인prod Admin8084
grafana.도메인Grafana3000

Nginx가 서브도메인 → 포트 매핑을 담당하니까, 앱 입장에서는 자기 포트만 신경 쓰면 된다.

핵심

  • 도메인: IP에 이름 붙여주는 것. 응답은 서버가 한다
  • Nginx: 안내데스크. Host 헤더 보고 적절한 서비스로 전달
  • 리버스 프록시: 클라이언트 → Nginx → 실제 서비스. 클라이언트는 뒷단 구조를 모른다
  • 서브도메인: 하나의 도메인으로 여러 서비스를 구분. 무제한 무료
