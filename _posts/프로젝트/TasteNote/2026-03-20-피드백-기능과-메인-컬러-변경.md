## 두 가지

[코치마크](/posts/코치마크-튜토리얼-구현/) 끝내고 오늘 한 건 두 가지. 메인 컬러 변경과 피드백 기능 구현. PR 2개 머지.

## 메인 컬러 변경 — PR #12

앱 메인 컬러를 토스 블루(`#3182F6`)에서 테라코타 오렌지(`#C1440E`)로 바꿨다. 앱 아이콘과 브랜드 통일.

### 토큰 한 곳만 바꾸면 된다

디자인 토큰 시스템의 장점이 여기서 나온다. `C.primary`를 바꾸면 22군데, 15개 파일에서 쓰는 곳이 자동으로 바뀐다. CSS 변수(`--color-primary`)도 같이 갱신. WCAG AA 대비비 4.87:1 확인.

`granite.config.ts`의 `primaryColor`도 동기화. 버전 트리의 `BRANCH_COLORS[0]`도 새 primary로 맞췄다 — 메인 브랜치(lane 0) 색상이 앱 primary와 다르면 어색하다.

### 리팩토링

바꾸면서 정리한 것들:
- `GRADIENT_BANNER` — 이름은 그래디언트인데 실제로는 primary 색상의 단일값. 삭제
- `C.indigo` — 쓰는 곳 없음. 삭제
- `BRANCH_COLORS` — 3개 파일에서 중복 정의. 1곳으로 통합

## 피드백 기능 — PR #13

[BE에서 Discord Webhook 연동을 했으니](/posts/홈서버-배포와-CI-CD-구축/) FE에서 피드백 폼을 만들었다. DB 없이 Discord 채널로 직접 전송하는 구조.

### FSD 구조

[FSD 학습 포스트](/posts/FSD-모듈-설계와-빌드타임-상수/)에 정리한 대로 `features/feedback/`에 모았다.

3개 커밋으로 나눠서 진행:
1. **인프라** — 라우팅, 타입, Vite 빌드타임 상수(`__APP_VERSION__`)
2. **핵심 모듈** — useSubmitFeedback 훅, collectDeviceInfo 유틸, MSW 핸들러
3. **UI + 통합** — FeedbackPage, 설정 메뉴 연결, Analytics 이벤트

### 디바이스 정보 자동 수집

피드백에서 중요한 건 디바이스 정보다. 사용자한테 "어떤 기기 쓰세요?"라고 물으면 제대로 된 답이 안 온다. `navigator.userAgent`를 파싱해서 OS, 버전, 디바이스 정보를 자동 수집한다. 앱 버전도 Vite `define`으로 빌드 타임에 주입해서 포함.

```typescript
const deviceInfo = collectDeviceInfo();
// { os: "iOS", osVersion: "17.4", device: "iPhone", appVersion: "1.2.3" }
```

### 설정 메뉴 연결

설정 페이지에 "문의/피드백" 메뉴 항목 추가. 피드백 페이지는 `AuthGuard` 적용 — 로그인한 사용자만 보낼 수 있다.

### MSW mock

실제 BE 배포 전이니까 MSW 핸들러로 목 처리. `POST /api/v1/feedback`에 대한 성공 응답 반환. BE 배포 후 MSW를 제거하면 바로 실제 API로 전환된다.

## Claude Code 협업

메인 컬러 변경은 "primary 색상을 이걸로 바꿔"라고 한 줄이면 끝이다. Claude가 토큰 파일부터 CSS 변수, config까지 전부 찾아서 바꿨다. `GRADIENT_BANNER` 삭제나 `BRANCH_COLORS` 통합 같은 정리도 같이 해줬다.

피드백 기능은 "Discord Webhook으로 보내는 피드백 폼 만들어줘, FSD 구조로"라고 지시. 커밋도 인프라 → 핵심 → UI 순서로 나눠서 했다. `collectDeviceInfo`에서 UA 파싱하는 건 Claude가 알아서 구현. Vite `define`으로 앱 버전 주입하는 것도.

다만 `__APP_VERSION__`의 타입 선언을 `vite-env.d.ts`에 추가하는 건 내가 "타입 에러 안 나?"라고 물어서 추가된 거다.
