## "이어서 할까요?"는 뭘 이어서 하는 건데

[TasteNote 요리모드](/posts/요리모드-v2-구현과-UX-개선/)에서 `CookingResumeSheet`를 개선했다.

처음에는 이게 문제인 줄 몰랐다. 레시피가 1개일 때는 "진행 중인 요리가 있어요. 이어서 할까요?"가 모호하지 않다. 어차피 한 개니까.

V2에서 레시피가 여러 개가 됐다. A 레시피를 하다 B 레시피 상세로 이동하는 경우가 생겼다. 그 시점에 이 시트가 뜨면 어떤 레시피를 이어서 하는 건지 화면만 봐서는 모른다. "이어서"가 A를 뜻하는지 B를 뜻하는지.

## 처음 수정 시도

처음엔 메시지에 "가장 최근에 요리하던 레시피"라는 말만 추가하려 했다. "가장 최근에 요리하던 레시피를 이어서 할까요?"

근데 그게 뭔지 여전히 안 보인다. "가장 최근"이 A인지 B인지 사용자는 모른다.

레시피명을 직접 넣어야 했다.

## 역방향 설계로 이어진 이유

레시피명을 메시지에 넣으려면, 세션 시작 시점에 레시피명을 저장해야 한다. 안 하면 나중에 "표시하고 싶은데 데이터가 없다"가 된다. 스토어를 다시 뜯어야 한다.

이 과정에서 설계 습관이 생겼다. "뭘 보여줄까?" → "그러면 뭘 저장해야 해?" 순서로.

```typescript
// cookingSessionStore에 추가
recipeName: string | null;  // 재개 시트에 표시하기 위해

// 세션 시작 시
startSession({ recipeId, versionId, recipeName });
```

UI 요구사항에서 역으로 데이터 모델을 설계하는 습관. 화면 기획이 완성되기 전에 어떤 데이터가 필요한지 먼저 파악하면 뒤늦은 수정을 줄일 수 있다.

## 좋은 시스템 메시지의 3요소

**1. 상황**: 지금 무슨 일이 일어났나

> "진행 중인 요리가 있어요"

**2. 맥락**: 무엇에 대한 이야기인가

> "**된장찌개**를 하다 멈췄어요"

**3. 행동**: 각 선택지가 어떤 결과를 만드나

> "된장찌개 이어서 하기 / 김치찌개 새로 시작"

셋 중 **맥락**이 가장 자주 빠진다. "이어서 할까요?"는 상황+행동은 있지만 맥락이 없다.

## 버튼 레이블: "수정" → "수량변경"

이건 QA 중에 발견했다. 재료 영역 버튼이 "수정"이었는데, 직접 눌러보니 수량만 바뀌었다. 이름도 수정, 순서도 수정, 수량도 수정이 가능한 줄 알았는데 수량만 됐다. 버튼명이 범위를 암시해야 한다.

"수정"은 범위가 넓다. 사용자가 버튼을 눌렀을 때 무슨 일이 일어날지 예측하기 어렵다.

"수량변경"은 명확하다. 재료의 수량만 바꾼다는 걸 안다.

좋은 버튼 레이블 체크리스트:
- **동사 포함**: "수정"보다 "수량 변경하기"
- **행동 대상 명확**: "삭제"보다 "이 재료 제외"
- **가역성 암시**: "삭제"(되돌리기 어려움) vs "숨기기"(다시 볼 수 있음)

## 프로젝트 적용

```
Before: "진행 중인 요리가 있어요. 이어서 할까요?"
        [이어서] [새로 시작]

After:  "된장찌개를 하다 멈췄어요."
        "된장찌개 이어서 하기" / "김치찌개 새로 시작"
```

버튼 레이블에 레시피명을 넣었다. 사용자가 선택하기 전에 결과를 예측할 수 있다. 영화관에서 "A관 또는 B관?" 대신 "인셉션(A관) 또는 오펜하이머(B관)?"으로 물어보는 거다.
