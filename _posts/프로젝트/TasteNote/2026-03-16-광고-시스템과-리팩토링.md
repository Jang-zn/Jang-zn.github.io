## 광고를 넣어야 밥을 먹는다

[저번에](/posts/TasteNote-프로젝트-시작/) Phase 2까지 끝냈으니 이제 현실적인 문제. **돈**. 광고를 넣는다.

## AdAdapter 인터페이스 설계

광고도 Adapter 패턴으로 갔다. 토스 환경이면 **TossAds**, 웹이면 **GoogleAdMob**. 인터페이스 하나로 갈아끼울 수 있게.

```typescript
interface AdAdapter {
  showBanner(slot: string): void;
  showInterstitial(): Promise<void>;
  showRewarded(): Promise<{ rewarded: boolean }>;
}
```

피드에서 레시피 **6개마다 배너 광고** 하나씩 끼워넣는 방식으로 구현했다.

## "보상형 광고는 빼자"

처음에는 보상형(리워드) 광고도 넣으려고 했다. 사실 Claude한테 "광고 시스템 만들어줘" 하니까 배너, 전면, 보상형까지 전부 다 구현해서 가져왔다. 근데 생각해보니까 우리 앱에 리워드 시스템이 없다. 광고 보면 뭘 줄 건데? 레시피 하나 더 줘? 그건 좀 아닌 것 같아서 Claude가 만들어준 보상형 광고 코드를 **과감하게 전부 삭제**했다.

이게 Claude랑 작업할 때 중요한 포인트인데, 기술적으로 가능하다고 다 넣으면 안 된다. Claude는 시키면 다 만들어주니까, **"이거 진짜 필요한가?"** 판단은 내가 해야 한다.

오히려 이 삭제가 앱 품질을 올렸다고 생각한다. 없는 게 나은 기능도 있는 법.

## UI 리팩토링

광고 작업하면서 눈에 거슬리던 것들도 같이 정리했다.

### Stars → StarRating 교체

기존 `Stars` 컴포넌트가 구렸다. `StarRating`으로 교체. HeroImage랑 RecipeDetailPage 둘 다.

### 파비콘 SVG → PNG

파비콘이 SVG였는데 일부 브라우저에서 안 보인다. PNG로 바꿨다. 고민할 거 없다.

### 이미지 업로드 리팩토링

이미지 업로드 로직이 지저분했다. 파일 타입/사이즈 검증 추가, 이미지 수정 핸들러 신규.

## HeroImage 백버튼 제거

HeroImage에 뒤로가기 버튼이 있었는데 토스 앱에 네비게이션 바가 이미 있다. 중복. 제거.
