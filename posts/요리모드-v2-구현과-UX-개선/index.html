[TasteNote] 요리모드 v2 — 드래그&드롭과 인라인 편집

By 코드대장간
Posted Mar 26 6 min read

아침에 기반 구조, 오후에 뜯어고침

오전(08:03~10:58)에 요리모드 기반을 올렸다. 오후(13:31~16:53)에 절반을 다시 만들었다. 써보면 안 맞는 게 보인다.

오전 작업: cookingSessionStore, CookingPage 기반, 재료 체크, 단계 완료, CookingCompleteSheet, CookingResumeSheet, StarRatingInput. 기반이었던 CookingReviewSheet는 오전에 만들고 오후에 삭제.

오전 — 기반 구조

cookingSessionStore

Zustand + persist로 요리 세션을 localStorage에 저장한다. 앱을 껐다 켜도 요리 중이었으면 이어갈 수 있다. 자세한 건 학습 포스트.

24시간 TTL. 어제 시작한 세션은 오늘 자동 폐기.

CookingPage

Wake Lock으로 화면 꺼짐 방지. 요리 중에 화면이 꺼지면 손에 밀가루가 묻어있을 때 다시 켜기 어렵다. Wake Lock 학습 포스트.

세그먼트 탭(재료/단계) + 진행 바. FAB 메뉴에서 “밥해먹기” 버튼으로 진입.

재료 체크

처음에 IngredientAction(USED/ADJUSTED/SKIPPED) 순환으로 만들었다. 탭 한 번에 USED → ADJUSTED → SKIPPED → USED.

넛지 사이클

요리 완료 시 변경 사항이 있으면 “새 버전으로 저장할까요?” 넛지. 없으면 조용히 완료. 넛지사이클 학습 포스트.

오후 — 뜯어고침 6개

재료 레이아웃 2열

재료가 1열로 나열되면 스크롤이 길어진다. 2열 CSS Grid 카드로 변경. 재료 카드에 세그먼트 컨트롤 배치.

재료 체크 구조 변경

IngredientAction 순환을 버렸다. USED/ADJUSTED/SKIPPED를 하나의 순환으로 쓰면 “사용함”과 “수량 변경”이 너무 섞인다. 분리했다.

  • checkedIngredientIds: string[] — 체크 여부 (탭하면 토글)
  • ingredientAdjustments: Record<string, string> — 수량 변경값

재료 카드 상단에 원형 체크 아이콘, 하단에 “수량변경”/”패스” 칩 2개. Fitts’s Law — 크고 가까이 있는 타겟. 진입점 설계 학습 포스트.

“수량변경” 칩을 탭하면 인라인 수량 입력 확장. white-space: nowrap으로 텍스트 찌그러짐 방지.

단계 순서 변경 — 드래그&드롭

처음에 ▲▼ 버튼으로 순서를 바꿨다. 써보니 불편하다. 단계가 10개면 맨 아래 단계를 맨 위로 옮기려면 버튼을 9번 눌러야 한다.

Pointer Events 기반 드래그&드롭으로 교체. onPointerDown에서 드래그 시작, onPointerMove에서 위치 추적, onPointerUp에서 swapSteps(fromIdx, toIdx) 호출.

moveStep은 splice/insert 방식. 드래그 위치에 drop indicator 표시. 단계를 잡아서 원하는 위치로 끌어다 놓는다. 서랍 안에서 책을 꺼내서 원하는 자리에 꽂는 거다.

드래그 중 강조 표시(상단 굵은 선)가 산만해서 커밋 하나 후에 바로 제거.

이전 요리 재개 시트

CookingResumeSheet에 레시피명이 없었다. “요리를 하다 멈췄어요”만 나오면 어떤 레시피였는지 모른다.

cookingSessionStore에 recipeName 필드 추가. 재개 시트: “된장찌개를 하다 멈췄어요. 김치찌개를 새로 시작할까요?” 형식으로 변경.

방어 코드

hasModifications 계산에서 ingredientActions ?? {}와 steps ?? []로 방어. 세션 복원 시 필드가 없을 수 있다.

Claude Code 협업

오전 기반 구조는 “요리모드 전체 구현해줘, 스토어-페이지-컴포넌트 순으로”로 지시. 분량이 많아서 순서를 지정했다. 스토어 먼저 나와야 컴포넌트가 쓸 수 있으니까.

오후 개선은 각각 별도 지시. “재료를 2열 그리드로 바꿔”, “▲▼ 버튼을 드래그&드롭으로 교체해”, “재개 시트에 레시피명 추가해”.

드래그&드롭은 “Pointer Events 기반으로, 외부 라이브러리 없이”라고 조건을 달았다. react-dnd나 dnd-kit 없이 직접 구현. 라이브러리 하나 추가할 때마다 번들 크기가 늘어난다.

오늘의 결과

항목상태
cookingSessionStore (persist + TTL)완료
CookingPage (Wake Lock, 세그먼트 탭)완료
CookingIngredientList (2열 그리드, 체크/수량 분리)완료
CookingStepCards (단계 완료, 메모, 드래그&드롭)완료
CookingCompleteSheet (넛지 사이클)완료
CookingResumeSheet (레시피명 표시)완료
RecipeDetailPage 요리 진입점완료
