[TasteNote] 냉장고와 히스토리 — 3탭 네비게이션 완성

By 코드대장간
Posted Mar 26 6 min read

하단 탭 3개

요리모드를 만들면서 앱 구조를 바꿨다. 기존 단일 홈 화면에서 하단 탭 3개로 확장.

탭내용
레시피기존 홈 (레시피 목록)
재료냉장고
기록히스토리 (요리기록)

BottomTabBar 컴포넌트와 TabLayout을 새로 만들었다. useCurrentScreen 훅이 현재 pathname을 ScreenName으로 매핑해서 탭 활성 상태를 결정한다. 경로 하드코딩 제거. SCREEN_PATH 상수로 경로를 한곳에서 관리.

FAB(플로팅 버튼)이 하단 탭과 겹쳐서 z-index를 50에서 110으로, position: fixed, bottom: 72px로 수정.

PantryPage — 냉장고 관리

BE에서 오늘 Pantry API를 구현했다. FE가 바로 붙였다.

구성

  • 카테고리 칩 필터: 채소/육류/해산물/유제품/조미료/기타. 선택하면 해당 카테고리만 표시
  • PantryList: 카테고리별로 그룹핑. 각 그룹에 헤더
  • PantryItemRow: 재료명 + D-day 뱃지 (유통기한 설정 시)
  • AddIngredientSheet: 재료 검색 + 직접 추가

재료 직접 추가

검색 결과 하단에 + "키워드" 직접 추가 항목이 항상 보인다. 시스템에 없는 재료도 자유롭게 입력 가능. “고수”, “화자오”처럼 기본 재료 목록에 없는 것도 추가된다.

useCreateIngredient hook 신규. POST /api/v1/ingredients로 서버에 재료를 등록하고 pantry에 추가한다.

BE에서 Pantry가 name/category로 직접 저장하는 구조로 바뀐 것과 연동. FE도 ingredientId 대신 name + category를 전송한다.

hooks

hook역할
usePantryItems냉장고 목록 조회
useAddPantryItem재료 추가
useRemovePantryItem재료 삭제
useSearchIngredients재료 검색
useCreateIngredient재료 신규 생성
useCookableRecipes만들 수 있는 레시피 조회

HistoryPage — 요리기록

요리기록을 월별로 탐색하는 페이지.

구성

  • 월 네비게이션: < 2026년 2월 > 형식. 좌우 화살표로 월 이동. useMonthNavigation 훅
  • StatsCard: 해당 월 요리 횟수, 평균 별점, 총 소요시간
  • MonthlyTimeline: 날짜별로 묶인 요리기록 목록
  • CookingLogCard: 레시피명, 별점 (★ 표시), 메모, 소요시간. 탭하면 해당 레시피 상세로 이동

소요시간은 서버에서 ms로 내려주는데 FE에서 분 단위로 변환. formatDuration 유틸. “1시간 23분” 형식.

날짜 헤더는 formatDateHeader 유틸. “3월 26일 (수)” 형식.

hooks

hook역할
useCookingHistory월별 요리기록 조회
useCookingStats월별 통계 조회

공통 유틸 정리

페이지가 늘어나면서 중복 로직이 보였다. b76c028 커밋에서 일괄 추출.

  • useDebounce: 검색 입력 디바운스
  • Spinner, EmptyState: 공통 상태 UI
  • formatDateHeader, formatDuration: 날짜/시간 포맷
  • useMonthNavigation: 월 네비게이션 상태 관리

MSW 목 데이터

BE API가 아직 완전히 테스트되지 않아서 MSW로 개발했다. 냉장고 목록, 레시피 검색, 요리기록 월별 조회 등 핸들러 다수 추가. MSW 학습 포스트.

API 경로에서 /v1/ 중복이 있었다. baseURL이 /api/v1인데 hook에서 /v1/pantry로 적으면 /api/v1/v1/pantry가 된다. f680b55 커밋에서 9개 hook 경로 수정. 단순 실수지만 전부 404가 나오니까 바로 발견됐다.

Claude Code 협업

PantryPage와 HistoryPage 구현은 “BE API 스펙을 보고 FE 페이지 만들어줘”로 지시. BE PR의 스펙 문서를 같이 붙여줬다.

분량이 많아서 에이전트 2명 병렬 구성. PantryPage 담당, HistoryPage 담당. 공통 유틸은 따로 추출.

TabLayout의 스크롤 문제가 있었다. paddingBottom으로 잡으려다가 탭바에 컨텐츠가 가려지는 문제. flex column + overflow: auto로 구조 자체를 바꾸는 게 맞았다.

오늘의 결과

항목상태
BottomTabBar + TabLayout (3탭)완료
useCurrentScreen 훅완료
SCREEN_PATH 경로 상수화완료
PantryPage (카테고리 필터, 그룹 목록)완료
AddIngredientSheet + 직접 추가완료
HistoryPage (월 네비, 타임라인)완료
공통 유틸 6개 추출완료
MSW 핸들러 다수 추가완료
API 경로 /v1/ 중복 수정완료

V2 FE의 핵심 페이지가 완성됐다. 요리모드 → 기록 저장 → 히스토리에서 확인 → 냉장고 정리의 플로우가 연결됐다.
