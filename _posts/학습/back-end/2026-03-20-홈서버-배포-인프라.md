## 홈서버에 배포하기로 했다

[앱인토스-통합 Api](/posts/홈서버-배포와-CI-CD-구축/)에서 BE를 홈서버(Ubuntu)에 배포하기로 했다. AWS 비용 부담 없이 prod/dev 환경을 동시에 돌릴 수 있다. 대신 직접 챙겨야 할 게 많다. Docker 컨테이너화, 서버 보안, 외부 접근 경로.

## Docker 멀티스테이지 빌드

Spring Boot 앱을 Docker로 말 때 단일 스테이지로 빌드하면 JDK 전체가 이미지에 포함된다. 빌드 도구, 소스 코드, 테스트 파일 — 런타임에 필요 없는 것들이 수백 MB를 차지한다. 이사할 때 공구함까지 같이 보내는 꼴이다. 가구 조립 끝났으면 공구함은 두고 가야 한다.

멀티스테이지 빌드로 해결한다. 첫 번째 스테이지(JDK 21)에서 Gradle 빌드, 두 번째 스테이지(JRE 21)에서 결과 jar만 복사.

```dockerfile
FROM eclipse-temurin:21-jdk AS build
ARG MODULE=tastenote-api
COPY . .
RUN ./gradlew :app:${MODULE}:bootJar

FROM eclipse-temurin:21-jre
COPY --from=build /app/.../app.jar app.jar
```

`ARG MODULE`로 같은 Dockerfile에서 tastenote-api, admin-api 둘 다 빌드한다. 멀티모듈 프로젝트라 이게 필요했다.

## docker-compose — 10개 서비스

prod/dev 환경을 한 서버에서 분리해야 한다. docker-compose로 10개 서비스를 구성했다.

| 서비스 | 포트 | 역할 |
|--------|------|------|
| tastenote-api | 8080 | Prod API |
| admin-api | 8081 | Prod Admin |
| dev-tastenote-api | 8180 | Dev API |
| dev-admin-api | 8181 | Dev Admin |
| redis-prod | 6379 | Prod 캐시 |
| redis-dev | 6380 | Dev 캐시 |
| mysql | 3306 | DB (prod/dev DB 분리) |
| nginx | 80/443 | 리버스 프록시 |
| prometheus | 9090 | 메트릭 수집 |
| grafana | 3000 | 대시보드 |

MySQL init 스크립트로 prod/dev 데이터베이스를 자동 생성한다. 모든 서비스에 `restart: unless-stopped`와 로그 제한(10MB × 3) 적용.

## 서버 보안 하드닝

홈서버를 인터넷에 노출하면 기본 설정으로는 안 된다. 현관문을 열어두고 "도둑 안 들어오겠지"라고 생각하는 거랑 같다. 4가지 조치.

### SSH 키 인증

비밀번호 로그인을 끄고 SSH 키만 허용한다. `PasswordAuthentication no`, `PermitRootLogin no`. 비밀번호는 무차별 대입 공격에 취약하다. SSH 키는 사실상 불가능.

### UFW 방화벽

필요한 포트만 열고 나머지는 차단한다. SSH(커스텀 포트), HTTP(80), HTTPS(443)만 열었다. 기본 정책은 deny.

### fail2ban

SSH 로그인 실패가 반복되면 해당 IP를 자동 차단한다. 5회 실패 → 10분 밴. 무차별 대입 공격 방어.

### 자동 보안 업데이트

`unattended-upgrades`로 보안 패치를 자동 적용한다. 수동으로 매일 확인하는 건 현실적이지 않다.

## Cloudflare Tunnel

홈서버에서 가장 귀찮은 게 포트 포워딩이다. 공유기 설정, 동적 IP, ISP 차단 — 변수가 많다. Cloudflare Tunnel을 쓰면 이 문제가 사라진다.

`cloudflared`가 서버에서 Cloudflare로 아웃바운드 연결을 맺는다. 외부 요청은 Cloudflare → 터널 → 서버로 들어온다. 인바운드 포트를 열 필요가 없다. 집에 택배 수령함을 두는 대신, 내가 우체국에 가서 "이 주소로 오는 거 다 가져갈게"라고 하는 셈이다. 집 주소(IP)를 노출할 필요가 없다.

```
사용자 → Cloudflare CDN → Tunnel → 홈서버(localhost:80)
```

DNS에 CNAME 레코드를 걸면 끝. SSL도 Cloudflare가 처리한다. systemd 서비스로 등록해서 서버 재시작 시 자동 연결.

## Nginx 리버스 프록시

Cloudflare Tunnel이 80번 포트로 들어오면 Nginx가 서브도메인별로 라우팅한다.

```nginx
server {
    server_name api.${DOMAIN};
    location / { proxy_pass http://tastenote-api:8080; }
}
server {
    server_name dev-api.${DOMAIN};
    location / { proxy_pass http://dev-tastenote-api:8180; }
}
```

`nginx.conf.template` + `envsubst`로 도메인을 환경변수로 주입한다. 하드코딩하면 도메인 변경 시 설정 파일을 직접 고쳐야 한다.

## mTLS 인증서 관리

[어제](/posts/Jackson-전략-변경과-mTLS/) 토스 API 연동을 mTLS로 전환했는데, Docker 환경에서 인증서 파일 접근이 문제다. Docker volume으로 호스트의 인증서 디렉토리를 컨테이너에 마운트한다. 이미지에 인증서를 넣으면 안 된다 — 이미지는 레지스트리에 올라가니까.

## 정리

홈서버 배포는 클라우드보다 손이 많이 간다. Docker 컨테이너화, 보안 하드닝, 네트워크 설정, SSL 처리 — 전부 직접 해야 한다. 대신 인프라를 처음부터 끝까지 이해하게 된다. 클라우드가 뭘 해주고 있었는지 체감이 된다.
